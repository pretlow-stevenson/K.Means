Loom™

A Protocol for Cross-Model Consensus

The current LLM paradigm treats frontier models as isolated oracles, creating silos of reasoning. Loom shifts the architecture from discrete inference calls to a unified conversational substrate, enabling heterogeneous models to operate as a coordinated, multi-agent system.

The Interoperability Layer

  • Normalized Messaging Interface: Every node in the system communicates via a unified schema.
  • State Synchronization: Rather than passing linear history, Loom™ maintains a shared context object. This ensures every model is contextually aware of the full conversational state.
  • Zero-Copy Context Routing: Efficiently manages turn-taking and message injection to minimize latency while maximizing the semantic density.

Collective Intelligence via Consensus Formation

  • Diagnostic Disagreement: Areas of divergent reasoning serve as high-signal diagnostic surfaces.
  • Weighted Consensus: By synthesizing the outputs of multiple independent architectures, the system generates a reliability metric.
  • Cross-Model Peer Review: Models critique and refine the reasoning of others.
